You can get a robust, filterable list of all caches you own in Project‑GC by using the standard filters plus (optionally) a saved Custom Filter and Virtual GPS export. [project-gc](https://project-gc.com/qa/?qa=19024%2Flist-caches-found-by-owner-finder)

## Quick way: filter by owner in a map tool

From the main page you already have open:

1. In the top menu, go to **Tools → Map not found caches** or **Tools → Map Compare** (either will list caches with a filter panel). [project-gc](https://project-gc.com/w/Main_Page)
2. In the left filter panel, look for the **Owner / Placed by** or **Cache owner** field (it may be under “Ownership filters”). Enter your geocaching username exactly as it appears on Geocaching.com (e.g., SummittDweller). [project-gc](https://project-gc.com/w/Filters)
3. Clear country/region if you want *all* of your hides; if that’s too many, you can restrict by country, region, or other filters like type or status. [project-gc](https://project-gc.com/Home/Membership)
4. Apply the filter; the map will refresh and the list below the map should now be all caches you own. [project-gc](https://project-gc.com/qa/?qa=19024%2Flist-caches-found-by-owner-finder)

At this point you have a robust “query” that you can tweak further with date, type, D/T, attribute filters, etc. [project-gc](https://project-gc.com/w/Attribute)

## More robust: Custom Filter + Virtual GPS

If you want to reuse the query and export to GPX/CSV, set up a Custom Filter and send the result to a Virtual GPS. This needs a paid Project‑GC membership (which you likely have). [project-gc](https://project-gc.com/w/Attribute)

1. In the top‑right user menu, click your username, then **Custom filters**. [project-gc](https://project-gc.com/w/Main_Page)
2. Create a new custom filter:
   - Add a condition like **Cache owner = your_username**.  
   - Add any other conditions you want (e.g., include disabled/archived, cache type, size, attributes). [project-gc](https://project-gc.com/w/Filters)
   - Save this filter with a name such as “All my hides”.  
3. Go back to **Tools → Map Compare** (or another map‑based tool), and in the filter panel choose your saved Custom Filter instead of manually entering the owner. [project-gc](https://project-gc.com/Home/Membership)
4. Once the list of your caches appears under the map, click the “select all” checkbox in the list and then click the big **Add** button to add them to a **Virtual GPS** list. [project-gc](https://project-gc.com/qa/?qa=30459%2Ffind-cache-by-cache-owner-name)
5. Open **Virtual GPS** from your user menu; from there you can:
   - Export your owned caches as **GPX**.  
   - Export to a **spreadsheet‑friendly format** for more analysis. [project-gc](https://project-gc.com/qa/?qa=24257%2Fcan-download-list-all-the-caches-visited-some-analysis-myself)

This Virtual GPS route is what most power users use when they talk about a “robust” query they can refine, reuse, and export. [project-gc](https://project-gc.com/qa/?qa=30865%2Farchived-caches)

## Example starting setup

As an example, a solid reusable “all my caches” filter might be:

- Cache owner = SummittDweller  
- Include: Active, Disabled, Archived (if you want lifecycle‑complete data)  
- Countries: blank (for worldwide)  
- Types: traditional, mystery, multi, etc., excluding events if you want only physical hides.  

Saved as a Custom Filter, you can drop this into Map Compare, add results to a Virtual GPS, and export whenever you need an updated list. [project-gc](https://project-gc.com/qa/?qa=30459%2Ffind-cache-by-cache-owner-name)

Would you rather consume the result in GPX (for GSAK/Garmin/etc.) or as CSV for spreadsheet/DB work?
